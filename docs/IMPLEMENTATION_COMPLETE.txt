HOW IT WORKS
================================================================================

FLOW DIAGRAM:
    Browser App (TypeScript)
           ↓
    WebProvider (provider.web.ts)
           ↓
    Web Worker (llm.worker.ts)
           ↓ (message passing)
    Wasm Runtime (lib.rs)
           ├─ init()
           ├─ load_model()
           ├─ generate()
           └─ embed()
           ↓ (wasm-bindgen exports)
    FFI Bridge (ffi.rs)
           ├─ init_context()
           ├─ completion()
           ├─ embedding()
           └─ (other functions)
           ↓ (extern "C" calls)
    Compiled llama.cpp
           ├─ Context management
           ├─ Model loading
           ├─ Token generation
           └─ Embedding computation
           ↓
    Generated Text / Embeddings

FEATURES:
- Real llama.cpp inference in browser via Wasm
- Multiple concurrent models (up to 5)
- Memory-aware admission control
- Unified TypeScript API
- Works alongside native iOS/Android
- OPFS file caching
- Structured error handling

BUILD INSTRUCTIONS
================================================================================

QUICK START:

1. Install prerequisites:
   - Node.js 18+
   - Rust 1.70+
   - wasm-pack: cargo install wasm-pack

2. Build with real llama.cpp:
   npm install
   LLAMA_WASM_EMBED_CPP=1 npm run build:wasm:embed
   npm run build:wasm:assets

3. Run tests:
   npm run test:pwa:smoke

DETAILED INSTRUCTIONS:
   See BUILD_WASM_GUIDE.md for complete step-by-step guide

BUILD OUTPUTS:
   dist/wasm/llama_engine.wasm (2-5MB)  - Compiled Wasm binary
   dist/wasm/llama_engine.js            - wasm-bindgen wrapper
   dist/wasm/llama_engine.d.ts          - TypeScript definitions
